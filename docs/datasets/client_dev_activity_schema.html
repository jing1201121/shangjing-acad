GitHub Client Development Activity Dataset Schema

This dataset is generated by a custom GitHub Data Collector that continuously retrieves and organizes public activity data from major blockchain-related open-source repositories (e.g., Ethereum, Solana, BNB Chain). The collector interacts with GitHub’s REST and GraphQL APIs to record commits, pull requests, reviews, issues, and user participation across repositories — producing a structured, relational CSV dataset suitable for quantitative and network analysis.

Each CSV file represents one table in the database and is automatically deduplicated and updated incrementally.

Collectively, these tables provide a comprehensive temporal record of open-source collaboration and technological innovation within decentralized ecosystems, supporting empirical analysis of developer networks, project evolution, and innovation dynamics.

Source Description

Ecosystem Representative Repositories Example
Ethereum ethereum/go-ethereum, ethereum/erigon, hyperledger/besu Core client implementations
Solana solana-labs/solana Consensus & runtime development
BNB Chain bnb-chain/bsc EVM-compatible execution client

These repositories are continuously crawled via GitHub’s REST and GraphQL APIs, with multi-token authentication and concurrent workers for stable and complete data retrieval.

1. *-commits.csv

Stores all commit records for each repository and time window.

Field Description
repo Full repository name (e.g., ethereum/go-ethereum)
sha Unique commit hash (primary key)
author_login GitHub login of the commit author
author_name Author name (from Git metadata)
author_email Author email (from Git metadata)
author_date Commit author timestamp (UTC)
committer_login Committer login (may differ from author)
committer_date Committer timestamp (UTC)
message Commit message text
files Semicolon-separated list of modified file paths (only if commit_files=True)

2. *-commit-files.csv

Provides file-level details for each commit, enabling fine-grained analysis of code changes.

Field Description
repo Full repository name
sha Associated commit hash
filename Modified file path
additions Number of lines added
deletions Number of lines deleted
status Change type (added / modified / removed / renamed)

3. *-prs.csv

Contains metadata for all Pull Requests created within the given period.

Field Description
repo Full repository name
number Pull Request number (unique within repo)
title Pull Request title
author_login Login of the PR author
author_association Author’s relationship to the repository (OWNER / MEMBER / CONTRIBUTOR / FIRST_TIMER / etc.)
createdAt PR creation timestamp
closedAt PR closing timestamp
mergedAt Merge timestamp (if merged)
mergedBy Login of the user who merged the PR
additions Total lines added in the PR
deletions Total lines deleted in the PR
changedFiles Number of files changed
isDraft Whether the PR is a draft
commitCount Number of commits in the PR
participants Semicolon-separated list of all PR participants (authors, reviewers, commenters)

4. *-pr-reviews.csv

Captures individual review actions linked to each PR.

Field Description
repo Full repository name
pr_number Pull Request number
reviewer_login Login of the reviewer
createdAt Timestamp of the review
state Review state (APPROVED / COMMENTED / CHANGES_REQUESTED / etc.)
pr_author_login Login of the PR author (for relational mapping)

5. *-issues.csv

Lists issue-level discussions within each repository.

Field Description
repo Full repository name
issue_number Issue number
author_login Login of the user who created the issue
createdAt Issue creation timestamp
closedAt Issue closing timestamp
commenters Semicolon-separated list of all commenters on this issue

6. *-issue-commenters.csv

Provides a normalized one-to-many mapping between issues and their commenters.

Field Description
repo Full repository name
issue_number Associated issue number
commenter_login Login of the commenting user
commentedAt Comment creation timestamp

7. user-repo-activity.csv

Aggregated bipartite “user ↔︎ repository” interaction edges, useful for constructing developer–project networks.

Field Description
user_login GitHub login of the user
repo Full repository name
source Type of interaction (commit_author, commit_committer, pr_author, pr_participant, reviewer, issue_author, or issue_commenter)
